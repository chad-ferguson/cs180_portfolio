Overview

The goal of this project is to split Prokudin-Gorskii's glass plates into B/G/R channels, align the channels with x and y shifts, and stack them into a colored image.

Single-Scale (.jpg): For low-res images, I did a small windowed search over integer shifts and picked the alignment that best matches the interior of the image. This is applied to the three .jpg files in the given data.

Multi-Scale Pyramid (.tif): For the larger, high-res scans, I used the coarse-to-fine pyramid speed-up. This starts at a downsampled version to get a rough shift, then carries that estimate up the pyramid and refines it with tiny local searches, recursively calling the single-scale approach. For the final code (used for the images below), I used gradients instead of raw pixels to deal with misalignment that is discussed at the bottom of this page. This is applied for all of the various .tif files in the given data and chosen from the archive.

Troubleshooting

While all of the other .tif images worked fairly well with the pyramid approach on raw pixels as features, I had trouble trying to get emir.tif aligned. As mentioned in the spec, the brightness difference between channels might be causing the problems. I decided to swap to gradient magnitude instead of raw pixels, which fixed this misalignment. The image before and after the feature update are shown below.

I think this technically counts as bells and whistles, and it did give slightly better results for the other images, but I genuinely just wanted to get all of the alignments working for the files in the examples folder. I guess this is both troubleshooting and bells and whistles.
